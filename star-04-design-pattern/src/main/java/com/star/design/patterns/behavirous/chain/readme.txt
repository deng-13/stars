1. 定义：
  使多个对象都有机会处理请求，从而避免了请求的发送者和接受者之间的耦合关系，
  将这些对象连成一条链，并沿着这条链传递该请求，直到有对象处理它为止
2. 两个重要的 角色
  1- 抽象处理者 ：
    一：定义一个请求的处理方法 handleMessage，唯一对外开放的方法
    二：定义一个链的编排方法 setNext，设置下一个处理者
    三：定义具体的请求必须实现的两个方法：
      1> 定义自己能处理的级别 getHandlerLevel
      2> 具体的处理任务echo
  2- 具体处理者 ：
3. 优缺点
  优点：
    1- 将请求和处理分开。
      请求者可以不用知道是谁处理的，处理者可以不用知道请求的全貌，两个解耦，提高系统灵活性
  缺点：
    1- 性能问题，每个请求都是从头遍历到尾，链比较长的时候，是个问题
    2- 调试不方便
