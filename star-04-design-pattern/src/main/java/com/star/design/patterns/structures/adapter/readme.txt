1. 定义：
  将一个类的接口变成客户端所期待的另一种接口，从而使原本因接口不匹配而无法在一起工作的两个类能够在一起工作
2. 四个重要的 角色
  1- 目标角色 ： 我们所期望的接口，例子中的 IUserInfo
  2- 源角色 ： 想把谁转换成目标角色 的谁
  3- 适配器角色 ：适配器模式的核心角色，其他两个角色是已经存在的角色，适配器角色是新创建的
    职责就是 将源角色转换成目标角色
    转换的方式：1> 继承   2> 类关联

3. 优缺点
  优点：
    1- 可以让两个没有什么关系的类一起运行
    2- 增加了类的 透明性
    3- 提高了类的 复用度
    4- 灵活易用，不需要适配器的时候，删除就可以了。基本上类似一个灵活的构件
