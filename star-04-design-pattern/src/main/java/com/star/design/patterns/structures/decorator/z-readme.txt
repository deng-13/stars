1. 定义：
  动态地给 一个对象添加一些额外的职责
2. 四个重要的 角色
  1- 抽象构件 ： 是一个抽象类或是接口，定义了业务最核心的对象，比如成绩单
  2- 具体构件 ： 是抽象构件的 实现类，比如四年级的成绩单
  3- 装饰角色 ： 一般是抽象类？作用是什么呢-> 实现接口或抽象类
  4- 具体装饰角色
3. 优缺点
  优点：
    1- 装饰类和被装饰类可以独立发展，而不会相互耦合
    2- 装饰模式 是继承模式的 一种替代方案
    3- 可以动态的 对一个类进行扩展
  缺点：
    多层的装饰 会导致系统 越发的复杂
4. 使用场景
  1- 需要给一个类增加附加功能的时候
  2- 需要给 一批兄弟类改装或加强功能的时候
