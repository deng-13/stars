1. 定义：
  定义一个操作中的算法的框架，而将一些具体实现延迟到子类中。
  使得子类可以不改变一个算法结构的即可重新定义该算法的某些特定步骤
2. 两个重要的 角色
  1- 基本方法 ： 也叫基本操作，石油子类实现的方法，并在模版方法中被调用
  2- 模版方法 ： 可以是一个或几个，一般是一个具体方法，实现对基本方法的调度，完成固定的逻辑
  注：为防止恶意的操作，一般模版方法都加上final，不允许被覆写
3. 优缺点
  优点：
    1- 封装不变部分，扩展可变部分
    2- 提取公共代码，便于维护
    3- 行为由父类控制，子类实现
  缺点：
    ？
4. 使用场景
  1- 多个子类有公有的方法，并且逻辑基本相同时
  2- 重构时，模版方法可以把相同的代码抽取到父类，然后通过钩子函数约束其行为
